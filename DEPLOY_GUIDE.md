## 部署步骤

### 步骤 1：安装云函数依赖

在微信开发者工具中：

1. **展开云函数文件夹**
   - 在左侧文件树中找到 `cloudfunctions/`
   - 展开查看所有云函数

2. **为每个云函数安装依赖**
   
   对每个云函数（login、getUserInfo、createInvite 等）执行：
   
   **方法 A：右键安装（推荐）**
   - 右键点击云函数文件夹（如 `login`）
   - 选择 **"创建并部署：云端安装依赖"**
   - 等待部署完成
   
   **方法 B：终端安装**
   - 右键点击云函数文件夹
   - 选择 **"在外部终端窗口中打开"**
   - 运行命令：
   ```bash
   npm install
   ```
   - 然后右键云函数 → **"创建并部署：云端安装依赖"**

---

### 步骤 2：检查依赖安装

安装成功后，云函数文件夹应该包含：
```
cloudfunctions/login/
├── index.js
├── package.json
└── node_modules/        ← 这个文件夹应该存在
    ├── wx-server-sdk/
    └── ...
```

---

### 步骤 3：部署云函数

**单个部署：**
- 右键点击云函数文件夹
- 选择 **"创建并部署：云端安装依赖"**
- 等待控制台显示 "创建云函数成功"

**批量部署（如果单个失败）：**
1. 打开微信开发者工具的终端（菜单栏 → 工具 → 终端）
2. 运行：
```bash
# 进入项目目录
cd cloudfunctions/login
npm install

# 然后对每个云函数重复...
```

---

### 步骤 4：验证部署

1. **查看云开发控制台**
   - 点击微信开发者工具上方的 "云开发" 按钮
   - 进入 "云函数" 标签
   - 应该能看到已部署的函数列表

2. **测试云函数**
   - 在云函数列表中点击某个函数
   - 点击 "测试" 按钮
   - 查看是否能正常执行

## 常见问题

### 问题 1：右键没有 "创建并部署" 选项

**原因：** 微信开发者工具没有识别为云函数项目

**解决：**
1. 检查 `project.config.json` 是否有以下配置：
```json
{
  "cloudfunctionRoot": "cloudfunctions/"
}
```

2. 重新导入项目：
   - 项目 → 重新打开此项目

### 问题 2：部署失败，提示 "没有找到 node_modules"

**解决：**
```bash
# 在云函数目录下手动安装
cd cloudfunctions/login
npm install wx-server-sdk
```

### 问题 3：部署成功但调用失败

**检查：**
1. 云开发环境ID是否正确
2. 数据库集合是否已创建
3. 数据库权限是否设置

1. **users** - 用户表
2. **invites** - 邀请码表
3. **letters** - 信件表
4. **streaks** - 连续打卡表
